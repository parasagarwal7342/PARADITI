## 📄 Part 2: Patent Claims (Novelty & Inventive Step)
These claims describe the "Novelty" and "Inventive Step" of P Λ R Λ D I T I, protecting the *logic* rather than just the code.

### Claim A: Multi-Path Self-Healing Retrieval with Zero-Downtime Fallback
**Description**: An autonomous system for mitigating government portal "Link Rot" and API instability.
- **Novelty**: Combining real-time health-checking with an "Active-Active" data strategy that hot-swaps between unstable official APIs and a cryptographically-verified local "Fallback Mesh" to ensure 100% scheme availability.

### Claim B: The "Welfare Dependency Graph"
**Description**: Modeling government schemes as a Directed Acyclic Graph (DAG).
- **Novelty**: Performing topological sorts on scheme nodes based on dynamic user eligibility attributes to generate an "Ordered Roadmap" of prerequisites.

### Claim C: The "Immutable Integrity Ledger"
**Description**: A system for generating non-repudiable application receipts without a distributed blockchain.
- **Novelty**: Utilizing a SHA-256 chained hash ledger hosted on a centralized server to provide mathematical proof of transaction sequence and payload integrity.

### Claim D: Dynamic WAF-lite Firewall
**Description**: Application-level security specifically tailored for welfare data.
- **Novelty**: A targeted pattern-matching firewall that intercepts and logs malicious injection attempts (SQLi/XSS) specifically within the context of government application form data.

### Claim E: Federated Data Orchestration & Normalization for GovTech
**Description**: High-fidelity automated form filling and data merging across heterogeneous sources.
- **Novelty**: A system that orchestrates and normalizes data from API Setu, myScheme, User Profile, and OCR-Extracted Document Data into a unified internal schema while automatically assigning "Trust Badges" to officially verified records.

### Claim F: Universal Beneficiary Score (UBS) Engine
**Description**: A quantification engine that converts fragmented personal data into a "Welfare Credit Score."
- **Novelty**: An algorithmic system that performs multi-variate analysis of citizen document authenticity and profile completeness to predict and maximize benefit absorption.

### Claim G: Intelligent Document Orchestration (IDO)
**Description**: An agentic document entry system that acts as a "Universal Adapter" for government portals.
- **Novelty**: A middleware that automatically detects strict compliance requirements (e.g., "Max 200KB JPEG", "Grayscale Only") of target government portals and autonomously recompresses, resizes, and reformats citizen documents on-the-fly to ensure 100% acceptance rates without user intervention.

### Claim H: Semantic One-Click Filing Agent
**Description**: A generative agent that maps unstructured personal data to structured government forms.
- **Novelty**: Utilizing Optical Character Recognition (OCR) combined with fuzzy logic to extract validated fields from identity documents (Aadhaar/PAN) and autonomously populate heterogeneous application forms (ApplicationService), creating a "Zero-Entry" submission flow.

### Claim I: Dialect-Adaptive Intent Mapper
**Description**: A voice-first AI system that translates colloquial regional dialects into structured welfare intents.
- **Novelty**: An NLP layer that bridges the gap between rural speech patterns (e.g., Bhojpuri, Magahi) and standard government nomenclature, enabling non-literate users to discover benefits through natural speech.

### Claim J: Asynchronous Benefit Queue & Hub
**Description**: A multi-channel application gateway for low-connectivity environments.
- **Novelty**: A system that allows citizens to initiate complex applications via SMS or WhatsApp (Offline-First), queuing them in a "Benefit Buffer" that syncs and processes full document packets once a stable connection is established.

### Claim K: Predictive Rejection Engine
**Description**: A pre-submission compliance model trained on historical rejection data.
- **Novelty**: An ML model that identifies "High-Risk" application triggers—such as document expiration or mismatching data—BEFORE submission, providing a predictive success probability score.

### Claim L: Autonomous Appeal Generator
**Description**: A legal-agent subsystem for automated reconsideration of denied claims.
- **Novelty**: An AI that monitors application status and, upon rejection, generates legally-sound appeal letters and RTI requests based on successful precedents and government guidelines.

### Claim M: Welfare-to-Work Sequencer
**Description**: A temporal logic engine for economic mobility pathway generation.
- **Novelty**: An AI that doesn't just match schemes, but *sequences* them (e.g., Educational Grant -> Skill Training -> MUDRA Loan) to create a personalized multi-year roadmap from welfare dependency to earning potential.

### Claim N: Impact ROI Calculator (Social Mobility Metric)
**Description**: A quantification engine for measuring the economic return of government investment in a citizen.
- **Novelty**: An algorithmic system that calculates the "Projected Income Increase" by correlating benefit absorption with employment outcomes, converting welfare into a measurable investment vehicle.

### Claim O: Programmable Benefit Tokenizer
**Description**: A financial layer for "Instruction-Led Spending" of welfare funds.
- **Novelty**: A system that issues smart-tokens (linked to e-RUPI) that can only be redeemed at verified vendors (e.g., fertilizer shops) based on the specific grant intent, eliminating leakage.

### Claim P: Decentralized Physical Triage Network (Agent Portal)
**Description**: A gig-economy framework for local assistance in offline zones.
- **Novelty**: A middleman orchestration layer that allows "Local Agents" to manage citizen portfolios, perform biometric verification, and earn success-based micro-commissions via the platform.

### Claim Q: Temporal Portal Arbitrage
**Description**: An autonomous queue manager for high-traffic gov-portals.
- **Novelty**: A system that monitors government server loads in real-time and auto-executes citizen submissions during low-traffic "Success Windows" to maximize approval rates.

### Claim R: Actuarial Social Risk Model (Social Credit Score)
**Description**: Using welfare history as a proxy for creditworthiness.
- **Novelty**: An ML model that converts UBS and transaction history into a "Financial Readiness Score," enabling banks to lend to unbanked individuals based on their government relationship history.

### Claim S: Geospatial Pre-emptive Trigger Logic
**Description**: Moving from "Reactive Application" to "Proactive Intervention."
- **Novelty**: A background engine that monitors external data streams (Satellite Weather Data, Mandi Prices, Local Disease Outbreaks). When a threshold is breached (e.g., Drought detected), it **autonomously** initializes relevant relief schemes (e.g., Crop Insurance) for all affected users in that geofence, requiring only a simple "One-Click Confirmation" from the citizen.

### Claim T: Zero-Knowledge Sovereign Agent (Edge AI)
**Description**: Privacy-first decentralized identity.
- **Novelty**: An architecture where the Aditi AI model runs locally on the user's device. It "negotiates" eligibility with the central server using Zero-Knowledge Proofs (ZK-SNARKs), proving the user is eligible (e.g., "Age > 18") without ever revealing the raw underlying data (DOB/Name) to the cloud, solving the "Surveillance State" paradox.

### Claim U: Peer-to-Peer Social Impact Bonds
**Description**: Privatized execution of public welfare.
- **Novelty**: A micro-market where private citizens can fund the "Gap Amount" for a beneficiary (e.g., paying the remaining 10% fees for a scholarship student). In return, the funder receives "Social Credit Tokens" or tax-offset certificates, creating a decentralized capitalization layer for the poor.

### Claim V: Hybrid State Security Mesh (HSSM)
**Description**: A fault-tolerant security architecture that maintains defensive integrity during infrastructure failures.
- **Novelty**: A dynamic "Fail-Safe" mechanism that instantly shifts from distributed Redis state-management to localized, thread-safe in-memory cryptographic enforcement upon failure detection. This prevents standard "Fail-Open" vulnerabilities where security layers degrade when dependencies (like cache servers) go offline, ensuring 100% protection uptime.
